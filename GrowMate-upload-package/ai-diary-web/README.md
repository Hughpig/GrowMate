# AI 多层标签日记网页端

这是第一版可运行网页端，使用 Python 标准库提供 Web 服务，SQLite 做持久数据库，浏览器访问页面使用。它不是桌面 App，也不需要打包安装。

## 已实现

- 邮箱注册、登录、退出。
- 多用户数据隔离：日记、标签、日程均按 `user_id` 查询。
- 纯文字输入限制：空内容、图片、表情、超过 2000 字会被拦截。
- 敏感词拦截：后端读取 `config/sensitive_words.txt`，命中后拒绝保存，避免违法违规、政治敏感或不符合本地风土人情的内容进入系统。
- 断网/提交失败策略：前端点击保存后立即清空输入框，不缓存草稿、不自动重试。
- SQLite 持久化：数据库文件位于 `data/app.db`，服务重启后数据保留。
- 日记保存后自动生成事件、情绪、性格、价值观、幸福度标签。
- 标签筛选：点击标签后筛选当前用户日记。
- 明显计划识别：识别到计划后弹窗询问是否加入日程。
- 日程确认创建：默认开始前 15 分钟提醒，可修改提醒选项。
- 可选 OpenAI 接入：设置 `OPENAI_API_KEY` 和 `OPENAI_MODEL` 后尝试调用 OpenAI；失败时记录 `analysis_jobs`。

## 运行

```powershell
python app.py
```

打开：

```text
http://127.0.0.1:8000
```

使用方式是网页端访问：服务器保持运行，用户通过浏览器打开网址录入日记。

## 可选环境变量

```powershell
$env:OPENAI_API_KEY="你的 key"
$env:OPENAI_MODEL="当前可用模型名"
$env:APP_TIMEZONE="Asia/Shanghai"
$env:PORT="8000"
python app.py
```

未设置 OpenAI 环境变量时，系统使用本地规则分析，方便开发和验收。

## 敏感词库

词库位置：

```text
config/sensitive_words.txt
```

维护方式：

- 一行一个词或短语。
- `#` 开头为注释。
- `re:` 开头表示正则规则。
- 修改后重启服务生效。
- 词库建议由管理员按当地法律法规、平台规则和实际使用场景维护，不建议放到前端。

## 验收示例

输入：

```text
今天和朋友出去逛街了，还吃了饭，工作日晚上下班无事
```

预期：保存原文，生成【社交、朋友、吃饭、工作轻松】以及情绪、性格、价值观、幸福度标签。

输入：

```text
明天下午三点和朋友看电影
```

预期：保存原文，弹出“加入日程？”确认框；确认后创建日程，默认提前 15 分钟提醒。

输入：

```text
这是一条包含政治敏感表达的测试
```

预期：拒绝保存，并提示内容包含敏感词或敏感表达。
