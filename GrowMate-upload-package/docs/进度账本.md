## S2 架构选型

### S2.1 当前状态

- 状态：已完成第一版可运行框架。
- 实现目录：`ai-diary-web`。
- 下一步：按需要升级为 React / NestJS / PostgreSQL / Redis 队列生产架构，或继续在当前 Python + SQLite 框架上补功能。

### S2.2 初步建议

- 前端：React + TypeScript + Vite。
- 后端：Node.js + NestJS 或 Fastify。
- 数据库：PostgreSQL。
- ORM：Prisma。
- 队列：BullMQ + Redis。
- 部署：Docker Compose 单机起步。
- 日程：独立 `schedules` 与 `schedule_reminders` 表，提醒任务由后台 worker 执行。

### S3.1 已完成

- Python 标准库 HTTP 服务，入口文件：`ai-diary-web/app.py`。
- SQLite 持久数据库，运行后生成：`ai-diary-web/data/app.db`。

- 日记纯文字输入限制、2000 字限制、表情和图片拦截。
- 服务端敏感词库拦截：`ai-diary-web/config/sensitive_words.txt`。
- 保存日记后自动生成事件、情绪、性格、价值观、幸福度标签。
- 标签筛选日记。
- 明显计划识别、确认后加入日程、默认提前 15 分钟提醒且可修改。
- 可选 OpenAI 接入：设置 `OPENAI_API_KEY` 和 `OPENAI_MODEL` 后启用，未设置时使用本地规则分析。
