# 产品架构与开发方案（可落地版）

> 项目代号：**GrowMate · 伴成长**  
> 文档用途：创业演示 / 立项 / 开发协作  
> 版本：MVP v0.1

---

## 1. 产品一句话

以**个人长期成长档案**为底座，以**AI 深度识人与陪伴**为内核，叠加体能 / 营养 / 心理 / 技术四大赋能体系，并提供垂直情绪与成长社区的一站式平台。

## 3. MVP 技术架构（已实现）

| 层级 | 选型 | 说明 |
|------|------|------|
| Web | Next.js 15 App Router | 页面 + API 一体化，利于快速演示 |
| 语言 | TypeScript | 全栈类型安全 |
| 样式 | Tailwind CSS 4 | 现代轻量 UI |
| ORM | Prisma | 可从 SQLite 平滑迁 PostgreSQL |
| DB | SQLite（开发） | 零运维启动；生产建议 PostgreSQL |
| 认证 | JWT + HttpOnly Cookie | 简单可靠 |
| AI | 规则引擎 + OpenAI 兼容接口 | 无 Key 可演示；有 Key 可升级真模型 |

## 6. AI 建模说明（当前实现）

MVP 采用**可解释规则引擎**：

1. 聚合最近日记、情绪、帖子文本
2. 关键词 + 统计特征提取标签（性格/优势/短板/目标）
3. 基于近 7 日情绪/压力生成 `normal | watch | alert`
4. 产出推荐与陪伴回复
5. 写入 `AiProfile` 供前端展示

当配置 `OPENAI_API_KEY` 后，陪伴对话自动走大模型，system prompt 注入用户画像。

## 7. 隐私与安全原则

1. 日记默认私密（`isPrivate=true`）
2. 情绪心理区默认鼓励匿名
3. JWT 仅存 HttpOnly Cookie
4. 危机关键词触发风险提示（非医疗诊断）
5. 生产环境必须：HTTPS、密钥轮换、敏感字段加密、审计日志

## 8. 开发阶段路线图

### Phase 1 — MVP（本仓库）

- 账号、日记、情绪、AI 档案、社区、课程、轻量监护

### Phase 2 — 运营可用

- 管理后台（课程/社区/用户/预警工单）
- 评论点赞、关注、通知
- 对象存储（图片）
- PostgreSQL + Redis
- 更完整的推荐系统与周报

### Phase 3 — 规模化

- 移动端（RN / 小程序）
- 向量检索（长期记忆）
- 实时消息
- 人工心理团队工作台
- 可观测性、风控、合规

## 10. 差异化壁垒（对外表述）

1. **先存档，后社交**：与娱乐社区逻辑反向
2. **双档案引擎**：成长档案 + 心理动态档案
3. **模块解耦**：社区与课程可独立运营与扩展
4. **AI + 人工兜底**：可演进为双重心理保障
